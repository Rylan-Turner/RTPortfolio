Project #1: Scene It Before

Summary

Scene It Before is a digital stage design tool intended for planning out the look of theatrical performances with real time collaboration in mind. I worked on this project with 6 people on a team(including me) during a semester of college. It consists of 3 main parts. These are a website to manage user functionality, a database to manage user relations to projects, and an editor portion built within Unity. The workflow is that users create their accounts and projects on the website and manage collaborators within their projects on the website as well(1st video). They are then able to choose a project to load which will then allow them to decorate a scene and others can load the project at the same time and be given all the updates that have happened since they were gone and see current updates in real time through websocket programming(2nd video). You can also switch which project you are currently in through the unity editor instead of having to go back to the website. We ended up with a product that is roughly 90% finished with the main updates needed being visual enhancements as it looks a little plain and some minor bugs need fixing.

Project #2: Client-Server Program within a Linux OS

Summary

This project had 2 installments. One was a program that handled server interactions with a single client. The 2nd installment handled server interactions with multiple clients. The interactions between clients and the server were handled with a FIFO pipe. The code is also displayed at the start of the recording if you would like to pause and take a look at that while I scroll through it. Also, to know which interface belongs to a client or the server you can see me execute the client program or the server program at the top of the CLI. (denoted by ./client or ./server)

In the first video it shows the server handling the single client and the calls I make to the server from the client which are:
1. Establish a connection.
2. Convert the given number into text.
3. Convert the given text into number.
4. Store something on the server. (in this case the word "Manatee")
5. Return the stored item to the client.
6. Terminate the client.
7. Connect to another client.
8. Terminate the server as well as the client.

In the second video it shows the server handling multiple clients with updated code from the singular client section and follows a similar scenario where I make these calls to the server:
1. Establish a connection from the first client window.
2. Establish a connections from the third client window.
3. Establish a connection from the second client window.
4. Convert number to text from the first client window.
5. Convert text to number from the second client window.
6. Store a word "StoredThingy" on the server from the third client window.
7. Retrieve "StoredThingy" and return it to the second client window.
8. Terminate the client belonging to the third client window.
9. Terminate the client belonging to the first client window.
10. Reconnect to the server from the first client window.
11. Reconnect to the server from the third client window.
(3:20 is the timestamp) In this section I then show off the file system of how the FIFO's are being created when a client connects and destroyed when a client exits. The FIFO belonging to the server is also destroyed when terminating the server along with the final client.
12. Terminate the client belonging to the first client window and observe the FIFO file also being deleted.
13. Terminate the client belonging to the second client window and observe the FIFO file also being deleted.
14. Call to terminate the server and the final client from the third client window and observe that the final client FIFO and the server FIFO are deleted.
